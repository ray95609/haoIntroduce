## ADDED Requirements

### Requirement: 展示教練核心資訊
網頁必須清晰展示教練姓名（林浩 Eric Lin）、職稱（XPORTS 肌力私人教練）以及其核心理念（「你需要的不是更努力，是有系統與專業的訓練方式。」）。

#### Scenario: 進入網頁查看核心資訊
- **WHEN** 使用者開啟網頁
- **THEN** 系統應在首屏展示教練姓名、職稱與醒目的標語

### Requirement: 展示專業領域項目
網頁必須包含三個核心專業領域：肌力訓練、長者肌力、專項運動，並附帶簡短說明。

#### Scenario: 查看專業項目列表
- **WHEN** 使用者向下捲動至專業項目區塊
- **THEN** 系統應以卡片或列表形式展示這三個項目及其說明（如「重塑體態線條」）

### Requirement: 展示實戰關鍵字標籤
網頁必須展示教練的專業標籤：練壯實戰、週期訓練、塑形雕塑、長者肌力。

#### Scenario: 識別教練特點
- **WHEN** 使用者瀏覽網頁內容
- **THEN** 系統應以藥丸型（Pill/Chip）標籤展示這四個關鍵字

### Requirement: LINE 預約導流
網頁必須包含 LINE 預約的 QR Code 圖片，並提供「立即預約體驗」的行動呼籲（CTA）。

#### Scenario: 使用者想要預約
- **WHEN** 使用者點擊預約按鈕或掃描 QR Code
- **THEN** 系統應引導使用者至 LINE 預約介面

### Requirement: 響應式佈局 (RWD)
網頁必須在手機、平板與電腦上都能正常顯示且不跑版。

#### Scenario: 手機瀏覽網頁
- **WHEN** 使用者使用手機開啟網頁
- **THEN** 系統應自動調整版面為單欄排列，確保文字易讀且圖片不超出螢幕

### Requirement: 單檔案靜態架構
網頁必須以單一 `index.html` 檔案形式存在，包含內嵌的 CSS 與 JS。

#### Scenario: 部署與分享
- **WHEN** 使用者直接開啟 `index.html` 檔案
- **THEN** 系統應能完整載入所有樣式與圖片連結，無需額外伺服器配置
